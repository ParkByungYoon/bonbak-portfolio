# YouTube 스타일 ML 엔지니어 포트폴리오 — 설계 문서

**작성일:** 2026-04-13  
**대상:** 머신러닝 엔지니어 취업준비생 포트폴리오 웹사이트

---

## 개요

유튜브 채널 페이지 UI를 그대로 재현한 정적 웹 포트폴리오.  
채널 메인 페이지에서 프로젝트 목록을 보여주고, 프로젝트 카드 클릭 시 상세 페이지로 이동한다.  
상세 페이지에서는 PDF.js로 PPT(PDF 변환본)를 슬라이드 형태로 표시한다.

---

## 기술 스택

- **언어:** 순수 HTML / CSS / JavaScript (빌드 도구 없음)
- **라이브러리:** PDF.js (PDF 렌더링)
- **배포:** GitHub Pages / Netlify / Vercel 등 정적 호스팅
- **로컬 실행:** `file://` 프로토콜로 바로 열기 가능

## 페이지 1: 채널 메인 페이지 (`index.html`)

### 레이아웃

```

│  사이드바 │  [아이콘] 채널명 · 직함 · 짧은 자기소개        │
│  (아이콘  ├──────────────────────────────────────────────┤
│  축소    │  탭: [홈] [동영상] [재생목록]                  │

│          ├──────────────────────────────────────────────┤
│          │  추천시스템 ──────────────────────────────    │
│          │  [썸네일카드] [썸네일카드] [썸네일카드]         │

│          │  [썸네일카드] [썸네일카드]                     │
│          │  멀티모달 ────────────────────────────────    │
│          │  [썸네일카드] ...                             │
│          │  머신러닝 ────────────────────────────────    │
│          │  [썸네일카드] ...                             │
└──────────┴──────────────────────────────────────────────┘
```

### 구성 요소

| 요소 | 설명 |
|------|------|
| 상단 헤더 | 유튜브 로고(포트폴리오 제목), 검색바(UI만), 알림/프로필 아이콘(UI만) |
| 왼쪽 사이드바 | 아이콘만 표시되는 축소 형태. 홈, Shorts, 구독, 보관함 등 유튜브 기본 아이콘 |
| 채널 아트 | 컬러 그라디언트 플레이스홀더. 나중에 이미지로 교체 가능한 구조 |
| 채널 정보 | 원형 프로필 플레이스홀더 + 채널명 + 직함 + 한 줄 자기소개 + 구독 버튼(UI만) |
| 탭 메뉴 | 홈: 대표 프로젝트 + 카테고리별 리스트 / 동영상: 전체 프로젝트 날짜순 그리드 / 재생목록: 카테고리별 그룹. 탭 클릭 시 페이지 이동 없이 콘텐츠 영역만 교체 |
| 대표 프로젝트 | `featured: true`인 프로젝트 1개. 큰 썸네일 + 제목 + 설명 |
| 카테고리별 리스트 | 추천시스템 → RAG → 멀티모달 → 머신러닝 순서로 가로 스크롤 카드 목록 |

### 프로젝트 카드

- 썸네일 이미지 (16:9 비율)
- 제목
- 카테고리 태그
- 간단 설명 (2줄 말줄임)
- 클릭 시 `video.html?id={project-id}` 로 이동

## 페이지 2: 프로젝트 상세 페이지 (`video.html`)

### 레이아웃

```
┌─────────────────────────────────────────────────────────────┐
│  상단 헤더 (동일)                                             │
├──────────┬──────────────────────────────────────────────────┤
│          │  ┌──────────────────────────┐  ┌──────────────┐  │
│  왼쪽    │  │                          │  │ 관련 프로젝트 │  │
│  사이드바 │  │    PDF 뷰어              │  │ ┌──────────┐ │  │
│  (동일)  │  │    (PDF.js 렌더링)       │  │ │[썸네일]  │ │  │
│          │  │  ◀ 1/12 ▶  [⛶전체화면]  │  │ │제목      │ │  │
│          │  └──────────────────────────┘  │ └──────────┘ │  │
│          │                                │ ...          │  │
│          │  프로젝트 제목                  └──────────────┘  │
│          │  채널명 · 업로드 날짜                              │
│          │  [👍 좋아요 N] [↗ 공유] [⬇ 저장]                 │
│          │  ──────────────────────────────                  │
│          │  프로젝트 상세 설명                                │
│          │  (사용 기술, 방법론, 성과 등)                      │
└──────────┴──────────────────────────────────────────────────┘
```

### 구성 요소

| 요소 | 설명 |
|------|------|
| PDF 뷰어 | PDF.js로 렌더링. 이전/다음 슬라이드 버튼, 페이지 번호 표시, 전체화면 버튼, 영화관 모드 버튼 |
| 영화관 모드 | 버튼 클릭 시 왼쪽 사이드바와 우측 관련 프로젝트 숨김. PDF 뷰어 영역 확장 |
| 좋아요 버튼 | 클릭 시 카운트 +1. `localStorage`에 저장 (새로고침 후에도 유지) |
| 공유 버튼 | 현재 페이지 URL을 클립보드에 복사. 완료 시 토스트 메시지 표시 |
| 저장 버튼 | 해당 프로젝트 PDF 파일 다운로드 |
| 프로젝트 정보 | 제목, 채널명, 날짜, 상세 설명, 기술 태그 |
| 관련 프로젝트 | 같은 카테고리의 다른 프로젝트를 우측 사이드바에 썸네일 카드로 표시 |

## 데이터 구조 (`data.js`)

```javascript
const PORTFOLIO_DATA = {
  channel: {
    name: "이름",
    handle: "@handle",
    description: "한 줄 자기소개",
    subscriberCount: "ML Engineer",  // 구독자 수 자리에 직함 표시
  },

  projects: [
    {
      id: "rag-chatbot",              // URL 쿼리스트링에 사용
      title: "RAG 기반 챗봇 시스템",
      category: "RAG",               // 추천시스템 | RAG | 멀티모달 | 머신러닝
      date: "2024-11",
      thumbnail: "assets/thumbnails/rag-chatbot.jpg",
      pdf: "assets/pdfs/rag-chatbot.pdf",
      description: "프로젝트 상세 설명...",
      tags: ["LangChain", "FAISS", "GPT-4"],
      featured: false,               // true면 채널 메인의 대표 영상으로 표시
      likes: 0,                      // 초기값 (실제 카운트는 localStorage 관리)
    },
  ]
};
```

**카테고리 표시 순서:** 추천시스템 → RAG → 멀티모달 → 머신러닝

## 라우팅

- 채널 메인: `index.html`
- 프로젝트 상세: `video.html?id={project-id}`
- `video.js`에서 `URLSearchParams`로 `id` 파라미터를 읽어 `data.js`에서 해당 프로젝트 데이터를 조회

---

## 좋아요 데이터 관리

- 키: `likes_{project-id}` (예: `likes_rag-chatbot`)
- 값: 숫자 (문자열로 저장, 읽을 때 `parseInt`)
- 초기값: `data.js`의 `likes` 필드
- 저장소: `localStorage` (기기/브라우저 로컬, 서버 동기화 없음)
